This example shows how to run image classification with a real ONNX model using tract. You will load MobileNet v2, preprocess an image with proper ImageNet normalization, run inference, and decode the top predicted class.

The full source lives in [`examples/onnx-mobilenet-v2`](https://github.com/sonos/tract/tree/main/examples/onnx-mobilenet-v2) in the tract repository.

## Prerequisites

<Steps>
  <Step title="Download the ONNX model">
    MobileNet v2 is available from the ONNX model zoo. Download the `mobilenetv2-7.onnx` file:

    ```bash
    wget https://s3.amazonaws.com/tract-ci-builds/tests/mobilenetv2-7.onnx
    ```
  </Step>

  <Step title="Get a test image">
    The example uses a portrait of Grace Hopper. Clone the tract repo and copy the image to your working directory, or supply any JPEG you like:

    ```bash
    git clone https://github.com/sonos/tract
    cp tract/examples/onnx-mobilenet-v2/grace_hopper.jpg .
    ```
  </Step>

  <Step title="Add dependencies to Cargo.toml">
    ```toml Cargo.toml
    [dependencies]
    tract = "*"
    image = "*"
    anyhow = "*"
    ```
  </Step>
</Steps>

## Full Rust source

```rust src/main.rs
use anyhow::Result;
use tract::prelude::*;

fn main() -> Result<()> {
    let model = tract::onnx()?
        .load("mobilenetv2-7.onnx")?
        .into_model()?
        .into_runnable()?;

    // Open image, resize it, and build a Tensor from it
    let image = image::open("grace_hopper.jpg").unwrap().to_rgb8();
    let resized =
        image::imageops::resize(&image, 224, 224, ::image::imageops::FilterType::Triangle);
    let input = tract_ndarray::Array4::from_shape_fn((1, 3, 224, 224), |(_, c, y, x)| {
        let mean = [0.485, 0.456, 0.406][c];
        let std  = [0.229, 0.224, 0.225][c];
        (resized[(x as _, y as _)][c] as f32 / 255.0 - mean) / std
    });

    // Run the model
    let result = model.run([input])?;

    // Find the class with the highest score
    let best = result[0]
        .as_slice::<f32>()?
        .iter()
        .zip(2..)
        .max_by(|a, b| a.0.partial_cmp(b.0).unwrap());
    println!("result: {best:?}");
    Ok(())
}
```

Run it with:

```bash
cargo run --release
```

Expected output:

```
result: Some((11.4773035, 654))
```

Class 654 in the ImageNet label list is `military uniform` — which is correct for Grace Hopper's portrait.

## Image preprocessing

MobileNet v2 expects a `[1, 3, 224, 224]` float tensor in **NCHW** layout (batch, channels, height, width).

The preprocessing pipeline is:

1. **Resize** the image to 224 × 224 pixels using bilinear filtering.
2. **Normalize** each channel from `[0, 255]` to `[0.0, 1.0]`, then subtract the ImageNet per-channel mean and divide by the standard deviation.

| Channel | Mean  | Std   |
| ------- | ----- | ----- |
| Red     | 0.485 | 0.229 |
| Green   | 0.456 | 0.224 |
| Blue    | 0.406 | 0.225 |

These values are the dataset statistics of ImageNet and are standard across most MobileNet and ResNet models trained on it.

<Info>
  tract re-exports `ndarray` as `tract_ndarray` so you always get the version that is compatible with tract's tensor conversions.
</Info>

## Interpreting the output

The model outputs a single `[1, 1000]` tensor of unnormalized logits, one per ImageNet class. To find the top prediction, take the argmax:

```rust
let best = result[0]
    .as_slice::<f32>()?
    .iter()
    .zip(2..)           // class indices start at 2 in this model variant
    .max_by(|a, b| a.0.partial_cmp(b.0).unwrap());
// best = Some((score, class_index))
```

Cross-reference the index against the `imagenet_slim_labels.txt` file to get a human-readable label.

## Python equivalent

<CodeGroup>

```python Python (tract bindings)
import tract
import numpy as np
from PIL import Image

model = (
    tract.onnx()
    .load("mobilenetv2-7.onnx")
    .into_model()
    .into_runnable()
)

img = Image.open("grace_hopper.jpg").convert("RGB").resize((224, 224))
arr = np.array(img, dtype=np.float32) / 255.0

mean = np.array([0.485, 0.456, 0.406], dtype=np.float32)
std  = np.array([0.229, 0.224, 0.225], dtype=np.float32)
arr = (arr - mean) / std

# HWC → NCHW
tensor = arr.transpose(2, 0, 1)[np.newaxis, :]  # shape: (1, 3, 224, 224)

result = model.run([tensor])
confidences = result[0].to_numpy()
print("class:", np.argmax(confidences))  # 652 → "military uniform"
```

## Dynamic batch sizes

A [second example binary](https://github.com/sonos/tract/blob/main/examples/onnx-mobilenet-v2/src/bin/dyn-shape.rs) in the same directory demonstrates symbolic dimensions for variable batch sizes. After setting a symbolic `N` on the batch axis, tract lazily discovers the batch size at runtime without requiring model re-optimization.

```bash
cargo run --bin dyn-shape
```

## Running on WebAssembly

The example also compiles to WASM with no code changes:

```bash
rustup target add wasm32-wasi
cargo build --target wasm32-wasi
wasmtime --dir . ../../target/wasm32-wasi/debug/example-onnx-mobilenet-v2.wasm
```

## Performance

<Tip>
  On a Raspberry Pi 3, tract runs MobileNet v2 significantly faster than TensorFlow Lite on the same hardware. On a Raspberry Pi Zero, Inception v3 runs in 39 s with tract versus 113 s with TF-Lite (single-core, Armv6).
</Tip>
