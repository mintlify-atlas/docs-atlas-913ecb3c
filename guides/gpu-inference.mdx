---
title: "GPU inference"
description: "Run tract models on CUDA or Metal GPU backends."
---

tract's execution model is built around a `Runtime` trait. The default runtime runs on the CPU, but pluggable backends allow the same `Model` to be executed on a GPU with minimal code changes.

<Warning>
  GPU runtimes are **experimental**. Not all operators are GPU-accelerated. Unsupported operators fall back to CPU execution transparently, but this can introduce extra copies and reduce throughput compared to a fully-fused GPU path. Test your specific model before relying on GPU inference in production.
</Warning>

## Available runtimes

<CardGroup cols={2}>
  <Card title="CUDA" icon="server">
    Linux and Windows. Requires the NVIDIA CUDA Toolkit and the `tract-cuda` crate.
  </Card>
  <Card title="Metal" icon="apple">
    macOS and iOS. Requires the `tract-metal` crate. No additional SDK installation needed — Metal is part of the Apple platform SDK.
  </Card>
</CardGroup>

## Setup

### CUDA (Linux / Windows)

Add `tract-cuda` alongside `tract` in `Cargo.toml`:

```toml Cargo.toml
[dependencies]
tract = "0.21"
tract-cuda = "0.21"
```

You must also have the [CUDA Toolkit](https://developer.nvidia.com/cuda-downloads) installed on the build and target machines.

### Metal (macOS / iOS)

Add `tract-metal` alongside `tract`:

```toml Cargo.toml
[dependencies]
tract = "0.21"
tract-metal = "0.21"
```

No additional SDK setup is needed on Apple platforms.

## Usage

<Steps>
  <Step title="Load and convert the model">
    Load and convert your model to a `Model` exactly as you would for CPU inference. The `Runtime` is selected afterward.

    ```rust
    use tract::prelude::*;

    let model = tract::onnx()?
        .load("model.onnx")?
        .into_model()?;
    ```
  </Step>

  <Step title="Select a runtime">
    Use `runtime_for_name` to obtain a named runtime. The name is resolved at runtime against linked backends.

    <Tabs>
      <Tab title="CUDA">
        ```rust
        use tract::prelude::*;

        let runtime = tract::runtime_for_name("cuda")?;
        ```
      </Tab>
      <Tab title="Metal">
        ```rust
        use tract::prelude::*;

        let runtime = tract::runtime_for_name("metal")?;
        ```
      </Tab>
      <Tab title="CPU (default)">
        ```rust
        use tract::prelude::*;

        let runtime = tract::runtime_for_name("default")?;
        ```
      </Tab>
    </Tabs>

    `runtime_for_name` returns an error if the requested backend is not available (e.g. calling `"cuda"` on macOS without the CUDA crate linked).
  </Step>

  <Step title="Prepare and run">
    Pass the `Model` to `runtime.prepare()` instead of calling `model.into_runnable()`.

    ```rust
    use tract::prelude::*;

    let model = tract::onnx()?
        .load("model.onnx")?
        .into_model()?;

    let runtime = tract::runtime_for_name("cuda")?;
    let runnable = runtime.prepare(model)?;

    let input = tract_ndarray::Array4::<f32>::zeros((1, 3, 224, 224));
    let result = runnable.run([input])?;
    ```
  </Step>
</Steps>

## Stateful execution

For models with internal state (recurrent layers, KV caches, streaming) use `spawn_state()` to create a persistent execution context and call `state.run()` per frame or token.

```rust
use tract::prelude::*;

let runtime = tract::runtime_for_name("metal")?;
let runnable = runtime.prepare(model)?;

// Create a persistent state — carries hidden activations between calls
let mut state = runnable.spawn_state()?;

for frame in audio_frames {
    let output = state.run([frame])?;
    process(output);
}
```

Each `State` is independent. You can spawn multiple states from the same `Runnable` for concurrent streams.

## Conditional compilation

The GPU crates use `extern crate` declarations gated by target platform in `api/rs/src/lib.rs`:

```rust
#[cfg(target_vendor = "apple")]
extern crate tract_metal;

#[cfg(any(target_os = "linux", target_os = "windows"))]
extern crate tract_cuda;
```

This means the correct backend is automatically linked based on the compilation target — no runtime feature flags required.

## Fallback behavior

<Info>
  When an operator is not implemented in the GPU backend, tract silently executes it on the CPU. The output is numerically correct, but performance may be lower than a fully-GPU path because of host-device data transfers.

  Use `runnable.cost_json()` to identify which operators dominate execution time and check whether they are GPU-accelerated in your installed version.
</Info>
