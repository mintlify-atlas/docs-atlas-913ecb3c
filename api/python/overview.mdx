tract is a neural network inference engine written in Rust. The Python bindings wrap the native library via ctypes, giving you direct access to tract's loading, cooking, and inference pipeline.

## Installation

```bash
pip install tract
```

## Import

```python
import tract
```

## Top-level functions

| Function | Returns | Description |
|---|---|---|
| `tract.onnx()` | `Onnx` | Create an ONNX loading context |
| `tract.nnef()` | `Nnef` | Create a NNEF loading/saving context |
| `tract.runtime_for_name(name)` | `Runtime` | Look up a named execution runtime |
| `tract.version()` | `str` | Return the native library version string |

## Quickstart

<Steps>
  <Step title="Load a model">
    Load an ONNX file and convert it to a runnable model in one fluent chain:

    ```python
    import tract
    import numpy

    model = (
        tract.onnx()
        .load("./mobilenetv2-7.onnx")
        .into_model()
        .into_runnable()
    )
    ```
  </Step>
  <Step title="Prepare inputs">
    Inputs are passed as a list of numpy arrays. tract converts them automatically:

    ```python
    grace_hopper = numpy.load("grace_hopper_1x3x224x244.npy")
    ```
  </Step>
  <Step title="Run inference">
    Call `run()` with the input list. Outputs are returned as a list of `Tensor` objects:

    ```python
    result = model.run([grace_hopper])
    confidences = result[0].to_numpy()
    assert numpy.argmax(confidences) == 652
    ```
  </Step>
</Steps>

## Working with numpy

tract inputs and outputs interoperate with numpy transparently:

- **Inputs** — pass `numpy.ndarray` objects directly to `run()`. tract converts them internally via `Tensor.from_numpy()`.
- **Outputs** — `run()` returns a list of `Tensor` objects. Call `.to_numpy()` on any of them to get a `numpy.ndarray` back.

```python
import numpy

# Input: numpy array -> Tensor (automatic)
input_array = numpy.random.rand(1, 3, 224, 224).astype(numpy.float32)
outputs = model.run([input_array])

# Output: Tensor -> numpy array
output_array = outputs[0].to_numpy()
```
