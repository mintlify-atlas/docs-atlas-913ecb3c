ONNX models are loaded through an `Onnx` context object. Because many ONNX models carry only partial shape and type information, they land as an `InferenceModel` rather than a fully-typed `Model`. You typically set the input facts and then call `into_model()` to proceed.

## Creating an Onnx context

```python
import tract

onnx = tract.onnx()
```

The context is lightweight; you only need one per loading operation and it can be discarded afterwards.

---

## Onnx methods

### `load`

```python
Onnx.load(path: str) -> InferenceModel
```

Load an ONNX model file from the given path.

```python
model = tract.onnx().load("./mobilenetv2-7.onnx")
```

### `load_buffer`

```python
Onnx.load_buffer(data: bytes) -> InferenceModel
```

Load an ONNX model from an in-memory bytes object. Useful when the model is bundled as a resource rather than a file on disk.

```python
with open("model.onnx", "rb") as f:
    data = f.read()

model = tract.onnx().load_buffer(data)
```

---

## InferenceModel methods

`InferenceModel` is the partially-typed form returned by the ONNX loader. Once input facts are set and shape inference runs, convert it to a `Model` with `into_model()`.

### Inspecting the model

| Method | Returns | Description |
|---|---|---|
| `input_count()` | `int` | Number of model inputs |
| `output_count()` | `int` | Number of model outputs |
| `input_name(id: int)` | `str` | Name of the `id`-th input |
| `output_name(id: int)` | `str` | Name of the `id`-th output |
| `input_fact(id: int)` | `InferenceFact` | Current fact for the `id`-th input |
| `output_fact(id: int)` | `InferenceFact` | Current fact for the `id`-th output |

### Setting facts

```python
InferenceModel.set_input_fact(id: int, fact: InferenceFact | str | None) -> None
InferenceModel.set_output_fact(id: int, fact: InferenceFact | str | None) -> None
```

Facts can be provided as:

- A **string spec** like `"1,3,224,224,f32"` â€” comma-separated dimensions followed by a type mnemonic (`f32`, `f16`, `i32`, `u8`, etc.)
- An **`InferenceFact`** object
- `None` to clear an existing fact

### Converting to a typed model

```python
InferenceModel.into_model() -> Model
```

Runs shape and type inference, then converts the `InferenceModel` to a fully-typed `Model`. The `InferenceModel` is consumed and becomes invalid after this call.

---

## Setting input shapes

Most ONNX models are exported with a symbolic batch dimension. You need to tell tract the concrete shape before converting.

```python
model = tract.onnx().load("./mobilenetv2-7.onnx")

# Set the first input to shape [1, 3, 224, 224] with float32 elements
model.set_input_fact(0, "1,3,224,224,f32")
```

The fact string format is: `dim0,dim1,...,dimN,dtype` where dtype is one of `f32`, `f64`, `f16`, `i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`.

## Examples

<CodeGroup>
```python Simple load and run
import tract
import numpy

model = (
    tract.onnx()
    .load("./mobilenetv2-7.onnx")
    .into_model()
    .into_runnable()
)

image = numpy.random.rand(1, 3, 224, 224).astype(numpy.float32)
result = model.run([image])
print(result[0].to_numpy().shape)
```

```python Explicit fact setting
import tract

# Load as InferenceModel
inference_model = tract.onnx().load("./model.onnx")

print(f"Inputs: {inference_model.input_count()}")
print(f"Input 0 name: {inference_model.input_name(0)}")

# Provide the shape/type for the first input
inference_model.set_input_fact(0, "1,3,224,224,f32")

# Convert to typed Model, then to Runnable
runnable = inference_model.into_model().into_runnable()
```

```python Load from bytes
import tract

with open("model.onnx", "rb") as f:
    data = f.read()

model = (
    tract.onnx()
    .load_buffer(data)
    .into_model()
    .into_runnable()
)
```
</CodeGroup>

<Note>
`into_model()` consumes the `InferenceModel`. Attempting to use it after conversion raises a `TractError`.
</Note>
